Rails ERD generates diagrams using Graphviz, a visualisation library. Continue to read how to install Graphviz and Rails ERD, so you can start creating model diagrams.

Install Graphviz

Rails ERD requires Graphviz (with Pango/Cairo support). To install it, try one of the following...

{% highlight console cssclass=console %} % brew install cairo pango graphviz # Homebrew on Mac OS X {% endhighlight %} {% highlight console cssclass=console %} % sudo port install graphviz # Macports on Mac OS X {% endhighlight %} {% highlight console cssclass=console %} % sudo aptitude install graphviz # Debian and Ubuntu {% endhighlight %}

Graphviz 2.22+ is recommended. Earlier versions may work, but many bugs have been fixed since. If you have trouble with earlier versions, you may wish to disable indirect relationships. Learn more.

Install Rails ERD

Next, add Rails ERD as a plugin to your Ruby on Rails 3 application. In your Gemfile...

{% highlight ruby %} group :development do gem "rails-erd" end {% endhighlight %}

Bundler will install everything you need...

{% highlight console cssclass=console %} % bundle install {% endhighlight %}

Generate your first diagram

You now have a new Rake task in your application. Generate an up-to-date model diagram with...

{% highlight console cssclass=console %} % rake erd {% endhighlight %}

Done! Open ERD.pdf to view your diagram. Not completely satisfied? You can customise the output.

Not using Rails?

If you use Active Record 3 outside of a Rails application, you can also use Rails ERD to create model diagrams for you. Install the rails-erd gem, load your models, and execute...

{% highlight ruby %} # Make sure all your models are loaded. require "rails_erd/diagram/graphviz" RailsERD::Diagram::Graphviz.create {% endhighlight %}
